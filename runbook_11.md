# Recover with get-wal

## Objective

Recover the latest base backup of the primary using `get-wal` as recovery options.


## Requirements:

- rb10


## Make sure Postgres is down and PGDATA is empty on the restore server

```bash
docker exec --user=postgres -i pgrestore bash << 'EOF'
  # CentOS 9 PGDG Path
  PGDATA_PATH="/var/lib/pgsql/17/data"
  PG_BIN="/usr/pgsql-17/bin/pg_ctl"

  # Stop and Clean
  $PG_BIN stop -D "$PGDATA_PATH" -m fast || echo "Postgres already stopped"
  rm -rf "${PGDATA_PATH:?}"/*
EOF
```


## Run barman recover

We have configured `recovery_options = 'get-wal'`:

```
barman@9c99447bf70f:~$ barman show-server rb08 | grep recovery_options
	recovery_options: RecoveryOptions({'get-wal'})
```

So we don't need to specify the `--get-wal` argument. Recovery would be simply:

```bash
barman recover \
  --remote-ssh-command "ssh postgres@pgrestore" \
  rb08 latest /var/lib/pgsql/17/data/
```

Barman has also created a file called `recovery.signal` and has filled recovery settings in `postgresql.auto.conf`:

```
[postgres@1b517f97b936 data]$ cat postgresql.auto.conf
# Do not edit this file manually!
# It will be overwritten by the ALTER SYSTEM command.
archive_mode = 'on'
#BARMAN#archive_command = 'barman-wal-archive pgbkp1 rb08 %p'
archive_command = false

# The 'barman-wal-restore' command is provided in the 'barman-cli' package
restore_command = 'barman-wal-restore -P -U barman 1c37152b8827 rb08 %f %p -p 4'
```

If it was PostgreSQL 11 or older, the recovery settings would have been written to the file `recovery.conf` instead.

We will need to amend the `restore_command`, replacing the container ID with the container name, so it's properly set to:

```
restore_command = 'barman-wal-restore -P -U barman pgbkp1 rb08 %f %p'
```


## Start Postgres

```bash
docker exec --user=postgres -i pgrestore /usr/pgsql-17/bin/pg_ctl start -D /var/lib/pgsql/17/data
```

Check Postgres ended recovery:

```
postgres@5333f7e45b0f:~$ psql -c 'SELECT pg_is_in_recovery()'
 pg_is_in_recovery
-------------------
 f
(1 row)
```
